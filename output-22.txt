There are a few steps involved in getting this to work. It should be something along these lines. The basic process is as follows:Create custom filters to check requests for specific authorization informationEach filter returns null (if no authorization of that type is found), or a custom AbstractAuthenticationTokenIf a filter returns a token, each AuthenticationProvider's supports(class) method will be invoked with that token returning true|false if it should try authenticationattemptAuthentication will then be called on the AuthenticationProvider which supports the token. Here you do any service calls to authenticate the user. You can then throw LoginException's or call authentication.setAuthenticated(true) and return the token for a successful authentication.I have been using this setup for a while supporting various authentication methods (signed request, username/password, oauth etc) and it works quite well. You can also pass AuthenticationSuccessHandler's and AuthenticationFailuersHandler's to the custom security filters to provide custom redirection strategies and failure handling.Also be sure to setup the ant matchers in the filter's constructors to control what url patterns the filters apply too. For example, an ldap request filter would probably need to be check with any request "/*" whereas a username/password filter can just be checked on POST's to /login or something similar.Example Code:1) Create custom AuthenticationToken's for each type of authentication you want to supportpublic class LDAPAuthorizationToken extends AbstractAuthenticationToken { private String token;
